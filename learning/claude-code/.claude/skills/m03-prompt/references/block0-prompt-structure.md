# Block 0: System Prompt의 구조

## Phase A — 개념

### 핵심 개념

**System Prompt** = 에이전트의 "두뇌". 같은 도구를 줘도 프롬프트에 따라 행동이 완전히 달라진다.

```
┌─────────────────────────────────────────────────┐
│           같은 도구 (Read, Glob, Grep)            │
├─────────────────────────────────────────────────┤
│  systemPrompt: "코드 리뷰어"  → 버그/보안 취약점   │
│  systemPrompt: "문서 작성자"  → README 개선 제안    │
│  systemPrompt: "아키텍트"     → 구조 개선 제안      │
└─────────────────────────────────────────────────┘
```

### 좋은 System Prompt의 3요소

```
1. 역할    — "당신은 ~입니다"
2. 관점    — "다음 기준으로 분석하세요"
3. 출력 형식 — "마크다운으로 출력하세요"
```

### 플러그인과의 연결

```
플러그인: agents/code-reviewer.md 의 본문 → Claude Code가 systemPrompt로 주입
SDK:     options.systemPrompt에 직접 문자열로 설정
```

플러그인에서 `.md` 파일에 적던 내용이 곧 systemPrompt다.

### 사전 질문

- "코드 리뷰어" 에이전트를 만든다면, System Prompt에 어떤 내용을 넣겠는가?
- 역할/관점/출력형식 3가지를 각각 한 문장으로 작성해보자

> ⛔ **STOP** — System Prompt 초안을 작성해서 공유해주세요.

## Phase B — 피드백 + 퀴즈

### 좋은 System Prompt 예시

```
당신은 시니어 TypeScript 코드 리뷰어입니다.           ← 역할
코드를 읽고 다음 관점에서 리뷰하세요:                  ← 관점
1. 에러 핸들링이 적절한가?
2. 타입 안전성은 확보되었는가?
3. 개선 제안 (구체적 코드 포함)

리뷰 결과를 마크다운 형식으로 출력하세요.              ← 출력 형식
```

### 퀴즈

**Q1**: systemPrompt를 생략하면 에이전트는 어떻게 행동하는가?

**Q2**: "당신은 코드 리뷰어입니다"만 쓰면 되는가, 아니면 관점과 출력 형식도 필요한가?

### 정답 가이드

- Q1: Claude Code의 기본 시스템 프롬프트가 적용됨 (범용 코딩 어시스턴트)
- Q2: 역할만으로도 동작하지만, 관점과 출력 형식을 추가하면 결과의 일관성과 품질이 크게 향상됨
