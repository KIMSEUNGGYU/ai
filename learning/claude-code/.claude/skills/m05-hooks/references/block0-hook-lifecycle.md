# Block 0: 훅 시점과 구조

## Phase A — 개념

### 핵심 개념

**Hooks** = 에이전트 루프의 **특정 시점에 개입**하는 콜백 함수.

```
사용자 프롬프트
       │
       ▼
  Claude 응답 → 도구 호출 요청
       │
  ┌────▼────┐
  │PreToolUse│ ← 도구 실행 전: 차단/수정 가능
  └────┬────┘
       ▼
    도구 실행
       │
  ┌────▼─────┐
  │PostToolUse│ ← 도구 실행 후: 감사 로그, 알림
  └────┬─────┘
       ▼
    결과 피드백 → 반복
       │
  ┌────▼──┐
  │ Stop  │ ← 에이전트 종료 시: 결과 검증, 정리
  └───────┘
```

### 훅 타입별 역할

| 훅 | 시점 | 용도 | 반환값 |
|----|------|------|--------|
| `PreToolUse` | 도구 실행 **전** | 차단, 입력 검증 | `{blocked: true}` 로 차단 |
| `PostToolUse` | 도구 실행 **후** | 감사 로그, 변경 추적 | `{}` (정보만 기록) |
| `Stop` | 에이전트 종료 | 결과 검증, 정리 작업 | `{}` |

### SDK 훅 vs 플러그인 훅

| 구분 | 플러그인 훅 | SDK 훅 |
|------|-----------|--------|
| 정의 | `.claude/hooks/*.json` + 셸 명령 | TypeScript 함수 |
| 로직 | 셸 스크립트 | 프로그래밍 (async, DB, API...) |
| 제어력 | 차단/허용 | 차단/허용 + 메시지 수정 + 비동기 |

### 훅 옵션 구조

```typescript
hooks: {
  PreToolUse: [
    { matcher: "Bash|Edit",  hooks: [myBlocker] }
  ],
  PostToolUse: [
    { matcher: ".*",         hooks: [myLogger] }
  ],
  Stop: [
    { matcher: ".*",         hooks: [myCleanup] }
  ],
}
```

`matcher`는 정규식으로 도구 이름을 필터링한다.

### 사전 질문

- 감사 로그에 어떤 정보를 기록하면 유용할까?
- `PreToolUse`에서 차단하면 에이전트는 어떻게 반응할까?

> ⛔ **STOP** — 위 질문에 대한 생각을 공유해주세요.

## Phase B — 피드백 + 퀴즈

### 피드백 포인트

- 감사 로그 필수 정보: 타임스탬프, 도구 이름, 입력 파라미터, 실행 결과
- PreToolUse 차단 시: 에이전트는 "도구가 차단되었습니다"라는 피드백을 받고, 다른 방법을 시도하거나 차단 이유를 사용자에게 보고

### 퀴즈

**Q1**: `matcher: ".*"`는 무엇을 의미하는가?

**Q2**: `PostToolUse` 훅에서 에이전트의 다음 행동을 변경할 수 있는가?

### 정답 가이드

- Q1: 모든 도구에 매칭. 도구 이름과 정규식 매칭
- Q2: PostToolUse는 도구 실행 **후**이므로 결과를 변경하기는 어렵지만, 로깅이나 알림은 가능. 행동 변경은 PreToolUse에서 해야 함
