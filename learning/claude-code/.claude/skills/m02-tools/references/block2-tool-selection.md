**실험 1**: `src/m02-tools/tools.ts`의 프롬프트를 다음으로 변경:

```typescript

```

**실험 2**: 프롬프트를 다음으로 변경:


## Phase B — 피드백 + 종합

### 핵심 인사이트

> **같은 도구라도 프롬프트에 따라 에이전트의 행동이 완전히 달라진다.**
> - "파일을 찾아서 읽어줘" → Glob → Read (탐색 → 확인)
> - "프로젝트 구조를 파악해줘" → Glob(반복) → Read(여러 파일)
>
> 도구는 "능력"이고, 프롬프트는 "의도"다. M03에서 `systemPrompt`로 의도를 더 정밀하게 제어한다.

### M02 종합 퀴즈

**Q1**: `tools: ["Read", "Glob"]`과 `tools: ["Read", "Glob", "Grep"]`의 실질적 차이는?

**Q2**: 에이전트가 도구를 **호출하지 않을 수도** 있는가? 어떤 경우에?

**Q3**: `tools`와 `allowedTools`를 동시에 설정하면 어떤 일이 발생하는가?

### 정답 가이드

- Q1: Grep이 추가되면 파일 내용 검색이 가능. "에러가 발생하는 코드를 찾아줘" 같은 패턴 검색 프롬프트에서 차이가 남
- Q2: 가능하다. "2+2는?" 같은 질문은 도구 없이 답변. 에이전트는 도구가 **필요하다고 판단할 때만** 호출
- Q3: `tools`로 사용 가능한 도구를 제한하고, 그 중에서 `allowedTools`로 자동 승인할 도구를 지정. `permissionMode: "bypassPermissions"`를 안 쓸 때 의미 있음
