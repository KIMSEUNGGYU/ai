# 트렌드 보고서 생성

## 입력
- `$ARGUMENTS`: 주차 번호 (예: `1`)

## 작업 절차

1. **소스 파일 읽기**:
   - `분석/` 폴더에서 가장 최신 트렌드 분석 파일을 읽는다
   - `정제/` 폴더에서 가장 최신 정제된 뉴스 파일을 읽는다
2. **보고서 템플릿 참조**: `템플릿/보고서_템플릿.md` 형식을 따른다
3. **보고서작성자 에이전트의 작성 원칙을 적용**:
   - 경영진 친화적 언어 (기술 용어에 괄호 설명 추가)
   - 결론 먼저, 숫자로 말하기, 액션 중심
   - 7건 뉴스 중 비즈니스 영향도 상위 3건 선별
   - 기술 트렌드 핵심 3-4개만, 경쟁사 3-4개만 요약
4. **보고서 저장**: `보고서/주간트렌드보고서_W{$ARGUMENTS}.md`로 저장

## 출력 형식

`템플릿/보고서_템플릿.md` 형식을 그대로 따르되:
- 핵심 트렌드 3줄은 비즈니스 임팩트 관점으로 작성
- 주요 뉴스는 상위 3건만 선별하여 시사점 포함
- 액션 아이템에 담당 부서와 기한을 구체적으로 제시
- 다음 주 관전 포인트는 2-3개로 압축

## 주의사항
- 간결하게 유지 (경영진이 5분 이내에 읽을 수 있는 분량)
- 불확실한 전망에는 근거를 함께 제시
