# 뉴스 정제 스킬

## 입력
- `$ARGUMENTS`: 정제할 뉴스 원본 파일 경로 (예: `수집/뉴스원본_20260131.md`)

## 작업 절차

1. **원본 파일 읽기**: `$ARGUMENTS` 경로의 뉴스 원본 파일을 읽는다
2. **템플릿 참조**: `템플릿/뉴스정제_형식.md` 파일의 형식을 따른다
3. **각 뉴스 항목마다 다음을 수행**:
   - **카테고리 분류**: 기술/시장/정책/기업/연구 중 택1
   - **핵심 내용 요약**: 원본 요약을 3문장 이내로 압축
   - **키워드 추출**: 핵심 키워드 3개 추출
   - **관련 기업/기관 정리**: 뉴스에 언급된 기업/기관 나열
4. **정제 파일 저장**: `정제/` 폴더에 `뉴스정제_YYYYMMDD.md` 형식으로 저장
   - 날짜는 원본 파일명의 날짜를 그대로 사용

## 카테고리 분류 기준

| 카테고리 | 기준 |
|---------|------|
| 기술 | AI 모델, 알고리즘, 아키텍처, 새로운 기술 발표 |
| 시장 | 시장 규모, 성장 전망, 투자, 경쟁 구도 |
| 정책 | 규제, 법률, 정부 정책, 윤리 가이드라인 |
| 기업 | 특정 기업의 제품 출시, 전략, 인수합병 |
| 연구 | 학술 연구, 논문, 실험 결과, 벤치마크 |

## 출력 형식

```markdown
# 정제된 뉴스 - YYYY-MM-DD

---

## 뉴스 N

- **제목**: {뉴스 제목}
- **출처**: {언론사명}
- **날짜**: {YYYY-MM-DD}
- **URL**: {링크}
- **카테고리**: 기술/시장/정책/기업/연구 중 택1

### 핵심 내용 (3문장 이내)
{압축된 요약}

### 키워드
- {키워드1}
- {키워드2}
- {키워드3}

### 관련 기업/기관
- {관련 기업/기관 목록}

---
```

## 주의사항
- 원본의 정보를 왜곡하지 않는다
- 카테고리가 애매한 경우 가장 핵심적인 주제 기준으로 분류한다
- 키워드는 구체적이고 검색 가능한 용어로 추출한다
