## Phase B — 피드백 + 종합

### 핵심 인사이트

> **System Prompt = 에이전트의 "세계관"**
> - 리뷰어: "이 코드에 버그가 있나?" 시선으로 봄
> - 멘토: "초보자가 이해할 수 있나?" 시선으로 봄
> - 보안 감사자: "공격 벡터가 있나?" 시선으로 봄
>
> 도구(능력)는 같지만, 프롬프트(관점)가 결과를 결정한다.

### M03 종합 퀴즈

**Q1**: System Prompt를 런타임에 동적으로 생성하는 것이 가능한가? 어떤 활용법이 있을까?

**Q2**: 플러그인의 `agents/reviewer.md` 파일과 SDK의 `systemPrompt` 옵션은 무엇이 같고, 무엇이 다른가?

### 정답 가이드

- Q1: 가능하다. TypeScript 코드이므로 조건에 따라 System Prompt를 조합할 수 있다. 예: 리뷰 대상 언어에 따라 다른 기준 적용, 프로젝트 설정에 따라 컨벤션 규칙 주입
- Q2: 같은 점 — 둘 다 에이전트의 역할/행동을 정의. 다른 점 — 플러그인은 .md 파일(정적), SDK는 코드(동적, 조건부, 변수 주입 가능)
