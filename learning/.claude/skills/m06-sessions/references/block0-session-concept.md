# Block 0: 세션 개념

## Phase A — 개념

### 핵심 개념

**Session = 에이전트의 기억 장치.** 대화 컨텍스트를 유지하고 이어갈 수 있다.

```
1차 쿼리: "인증 모듈을 읽어줘" → sessionId 캡처
                                      │
2차 쿼리: "그 모듈을 호출하는 곳은?"  ← resume: sessionId
         (1차 대화 맥락을 기억한 채로 답변)
```

### 세션이 없다면?

`query()`를 매번 새로 호출하면 에이전트는 **이전 대화를 모른다**.

```typescript
// 1차 쿼리 - 파일을 읽음
await query({ prompt: "package.json을 읽어줘" });

// 2차 쿼리 - "방금"이 무엇인지 모름!
await query({ prompt: "방금 읽은 파일의 의존성 수는?" });
// → "어떤 파일을 말씀하시는지 모르겠습니다"
```

### 세션의 실용적 활용

- **긴 작업 분할**: 1차에서 분석, 2차에서 수정, 3차에서 검증
- **피드백 루프**: 결과 확인 후 "이 부분 수정해줘"
- **세션 포크**: 같은 맥락에서 다른 방향 탐색
