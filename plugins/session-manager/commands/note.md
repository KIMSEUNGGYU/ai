# /note — 대화 학습 정리

대화에서 배운 것을 **나중에 다시 봤을 때 복습할 수 있는 형태**로 정리하여 저장한다.

## 핵심 원칙

이 스킬의 목적은 "대화 요약"이 아니라 **학습 추출**이다.

- **추출, 요약이 아님.** 대화 전체를 요약하지 않는다. "몰랐던 것 → 알게 된 것"만 뽑는다
- **자기완결적.** 대화 맥락 없이 노트만 읽어도 이해할 수 있어야 한다. "아까 말한 그것"은 금지. 독립된 문서처럼 작성
- **복습 최적화.** 핵심 → 상세 → 왜? 구조. 빠르게 훑거나 깊이 읽을 수 있다
- **선별적.** 대화에서 언급된 모든 것을 적지 않는다. 진짜 배운 것만 골라낸다

## 학습 추출 기준

대화를 스캔하여 아래에 해당하는 것만 추출한다:

| 유형 | 설명 | 예시 |
|------|------|------|
| 새로 알게 된 개념 | 몰랐던 것을 이해함 | "Suspense가 Promise를 throw해서 동작하는 거였어" |
| 교정된 이해 | 잘못 알던 것이 바로잡힘 | "useEffect cleanup이 unmount에만 도는 줄 알았는데 re-render마다 돈다" |
| 비직관적 인사이트 | 예상과 다르게 동작하는 것 | "StrictMode에서 effect가 2번 도는 건 버그가 아니라 의도" |
| 연결된 지식 | A와 B의 관계를 이해함 | "ErrorBoundary가 Suspense fallback도 같이 처리할 수 있구나" |

이미 알고 있었지만 대화에서 단순히 언급된 것은 **제외**한다.

## 동작

### 1. 정리 대상 파악

- 사용자가 명시하면 ("이 부분 정리해줘") 해당 내용에 집중
- 인자 없이 `/note`만 호출하면 → 대화 전체에서 학습 내용 자동 추출
- 추출 결과를 사용자에게 먼저 보여줌: "이런 내용을 정리하려 하는데 맞나요?"

### 2. 저장 위치 판단 + 확인

| 내용 성격 | 저장 위치 | 예시 |
|----------|----------|------|
| 개인 학습 (개념, 인사이트, TIL) | `~/hq/00_Inbox/{title}.md` | React 렌더링 이해, Git 전략 정리 |
| 프로젝트 설계/분석/결정 | `.ai/notes/{topic}.md` | API 구조 분석, 아키텍처 결정 |
| 재사용 코드 패턴 | `.ai/patterns/{name}.md` | 커스텀 훅 패턴, 에러 처리 패턴 |

판단 결과를 사용자에게 보여주고 확인:

```
학습 내용: React Suspense의 동작 원리와 ErrorBoundary 연동
저장 위치: ~/hq/00_Inbox/react-suspense-동작원리.md
이대로 저장할까요?
```

### 3. 기존 문서 확인

- Glob으로 유사 파일명 검색
- 기존 문서 있으면 → 업데이트 (새 내용 병합)
- 없으면 → 새로 생성

### 4. 저장

저장 완료 후:
```
/note 저장 완료 — {제목} → {저장 경로}
```

## 노트 형식

### ~/hq/ 학습 노트

Obsidian 노트로 저장. frontmatter 필수.

```markdown
---
tags: [learn, {주제 태그}]
date: {YYYY-MM-DD}
status: inbox
---

# {주제}

## 핵심 발견

- {배운 것 1} — {왜 중요한지 한 줄}
- {배운 것 2} — {왜 중요한지 한 줄}

## 상세

### {발견 1}

{자기완결적 설명. 대화 맥락 없이도 이해할 수 있게 작성.}

**왜?**
{근거, 이유, 원리. "왜 이렇게 동작하는가"에 대한 답}

{코드 예시가 도움이 되면 포함. 핵심 부분만, 전체 복붙 금지}

### {발견 2}

...

## 연관 키워드

- {나중에 검색할 때 걸릴 수 있는 관련 개념/기술}
```

**형식 원칙:**
- "핵심 발견"은 3~5개. 많으면 중요한 것만 선별
- 각 발견의 "왜?" 섹션이 노트의 핵심. 이유 없이 사실만 나열하지 않는다
- 코드는 핵심 부분만 인라인. 파일 전체를 복붙하지 않고 경로 참조

### .ai/notes/ 프로젝트 노트

```markdown
# {제목}

> 날짜: YYYY-MM-DD

## 요약
{한 줄 요약}

## 내용
{정리된 내용}

## 관련
- {관련 파일 경로 또는 참조}
```

### .ai/patterns/ 코드 패턴

```markdown
# {패턴명}

> 날짜: YYYY-MM-DD

## 언제 사용
{이 패턴을 적용할 상황}

## 패턴
{코드 또는 구조}

## 예시
{실제 사용 예}
```

## 원칙

- 저장 위치를 임의로 결정하지 말고 반드시 사용자에게 확인
- 기존 문서가 있으면 덮어쓰지 말고 병합
- 대화 내용을 그대로 옮기지 않는다. 정리하고 재구성한다
