작업 컨텍스트를 `.ai/active/{task}.md`에 저장한다. 세션 중 아무 때나 호출 가능.

$ARGUMENTS

## 동작

### 1. 작업 파일 결정

- 인자로 파일명이 주어지면 → `.ai/active/{파일명}.md` 사용
- 인자 없으면:
  1. `.ai/active/` 폴더 확인
  2. 파일이 1개면 → 그 파일 업데이트
  3. 파일이 여러 개면 → AskUserQuestion으로 어떤 작업인지 질문
  4. 파일이 없으면 → 대화 컨텍스트에서 작업 제목 추출 → 새 파일 생성

### 2. `.ai/active/` 폴더가 없을 때
→ 폴더 생성 후 진행

### 3. 해당 작업 파일이 없을 때
→ 대화 컨텍스트에서 아래 정보를 수집하여 새로 생성:

```markdown
# {작업 제목}

## 스펙
- {기획서 요약 or 링크}
- {주요 기능}

## 작업
- [x] {완료된 것}
- [ ] {남은 것}

## 현재 컨텍스트
{지금 하고 있는 작업의 구체적 상태}

## 결정사항
- {결정 + 이유}

## 기각된 대안
{논의했지만 채택하지 않은 접근법과 기각 이유. 없으면 이 섹션 생략}
- {대안} → 기각: {이유}

## 미해결 논의
{결론이 안 난 고민과 지금까지의 논의 맥락. 없으면 이 섹션 생략}
- {주제}: {어떤 논의가 있었고 왜 미결인지}

## 메모
{블로커, 주의사항 등. 없으면 이 섹션 생략}
```

### 4. 해당 작업 파일이 있을 때
→ 기존 내용을 읽고 섹션별 전략에 따라 업데이트:

**덮어쓰기 (최신 상태로 교체):**
- 작업 체크리스트 (완료 항목 체크, 새 항목 추가, 현재 상태 반영)
- 현재 컨텍스트 (최신 상태로 교체)
- 미해결 논의 (해결된 건 제거, 새로운 미결 추가)
- 메모 (블로커/주의사항 갱신)

**누적 (기존 내용 유지 + 새 항목 추가):**
- 결정사항 — 이전 세션의 결정을 절대 삭제하지 않는다. 새 결정만 아래에 추가
- 기각된 대안 — 이전 세션의 기각 기록을 절대 삭제하지 않는다. 새 기각만 아래에 추가

> 누적 섹션을 유지하는 이유: 다음 세션의 AI가 "왜 A 대신 B를 골랐는지" 알아야 같은 논의를 반복하지 않는다.

### 5. 세션 이력 기록
- 대화 컨텍스트에서 `[session-manager] session_id: {id}` 를 찾는다
- session_id가 있으면 → 작업 파일 끝의 `## 세션 이력` 섹션에 `- {session_id} ({timestamp})` 추가
- `## 세션 이력` 섹션이 없으면 → 파일 끝에 섹션 생성
- 이미 같은 session_id가 기록되어 있으면 → 건너뛴다
- timestamp 형식: `YYYY-MM-DD HH:MM` (현재 시각)

### 6. INDEX.md 동기화
- `.ai/INDEX.md`가 있으면 → "현재 진행 중" 섹션에 해당 작업 링크 추가/업데이트
- 없으면 → 생략 (INDEX.md는 별도 생성)

## 원칙
- 간결하게 (빠른 파악용)
- 코드 복붙 금지 (경로만 기록)
- **"왜" 필수** — 결정 이유, 기각 이유, 미해결 이유를 반드시 기록. "뭘 했는지"만 적고 "왜 그렇게 했는지"를 빠뜨리지 않는다
- **섹션별 전략** — 덮어쓰기 섹션은 최신 상태로 교체, 누적 섹션(결정사항/기각된 대안)은 기존 내용 유지 + 추가만
- 복잡한 작업은 작업 고유 섹션 자유롭게 추가
- active 파일 삭제 금지 — 삭제는 `/done`으로만 수행

## 대화 출력
저장 완료 후 한 줄 요약만 출력:
```
.ai/active/{파일명}.md 저장 완료 — {작업 제목} ({완료/진행중})
```
