# Claude 모델 호출 방식 비교

코드에서 Claude 모델을 호출하는 4가지 방식과 각각의 트레이드오프.

## 한눈에 보기

| | Plugin (Agent 도구) | `claude -p` (CLI) | Claude Agent SDK | Anthropic SDK (API) |
|---|---|---|---|---|
| 과금 | 구독 | 구독 | 구독 | 토큰 종량 |
| 호스트 | Claude Code | 내 코드 | 내 코드 | 내 코드 |
| 도구 | CC 내장 전부 | CC 내장 전부 | CC 내장 전부 | 직접 구현 |
| 제어 수준 | 프롬프트 | 프롬프트 in/out | 스트리밍, 세션 resume | 메시지 단위 완전 제어 |
| 컨텍스트 격리 | 부분적 (같은 세션) | 완전 (프로세스 분리) | 완전 (프로세스 분리) | 완전 |
| 컨텍스트 노이즈 | CC 시스템 프롬프트 주입 | CC 시스템 프롬프트 주입 | CC 시스템 프롬프트 주입 | 없음 (내가 보낸 것만) |
| 멀티턴 | 가능 (세션 내) | 불가 (1회 호출) | 가능 (session resume) | 가능 (대화 루프 직접) |
| 오버헤드 | 없음 (세션 내) | 매번 프로세스 스폰 | SDK가 관리 | HTTP 요청 |

## 1. Plugin (Agent 도구)

Claude Code **안에서** 서브에이전트로 실행. Claude Code가 호스트.

```
사용자 → Claude Code → Agent 도구 → 서브에이전트
```

- 실행: `/fe:harness` (Claude Code 세션 안에서)
- 장점: 별도 인프라 없음, 즉시 사용, CC 도구 전부 사용 가능
- 단점: 컨텍스트 격리 부분적 (같은 세션), 코드 제어 불가 (프롬프트 수준)
- 적합: 사용자와 인터랙티브하게 작업할 때, 빠른 프로토타이핑

## 2. `claude -p` (CLI 비대화형)

Claude Code CLI를 비대화형으로 실행. 내 코드에서 프로세스 스폰.

```typescript
execSync(`claude -p --model opus '프롬프트'`, { stdio: 'pipe' });
```

- 장점: 구독 과금 + 코드 제어 + CC 도구 사용
- 단점: 매 호출마다 프로세스 스폰 오버헤드, 멀티턴 불가, 블로킹
- 적합: 단순한 1회 호출, 빠른 구현

## 3. Claude Agent SDK (`@anthropic-ai/claude-agent-sdk`)

Claude Code CLI를 래핑하는 공식 SDK. async 스트리밍 지원.

```typescript
import { query } from '@anthropic-ai/claude-agent-sdk';

for await (const message of query({ prompt, options: { model: 'opus' } })) {
  // 스트리밍 메시지 처리
}
```

- 장점: 구독 과금 + 스트리밍 + 세션 resume + CC 도구 사용 + 서브에이전트 정의
- 단점: CC 시스템 프롬프트 노이즈는 여전히 존재
- 적합: **코드로 오케스트레이션하면서 구독 과금을 유지하고 싶을 때** (현재 fe-harness)

## 4. Anthropic SDK (API 직접 호출)

Anthropic API를 직접 호출. 완전한 제어.

```typescript
import Anthropic from '@anthropic-ai/sdk';
const client = new Anthropic();
const response = await client.messages.create({
  model: 'claude-sonnet-4-6-20250514',
  messages: [{ role: 'user', content: prompt }],
});
```

- 장점: 컨텍스트 노이즈 제로, 완전한 제어 (streaming, tool_use, 구조화 출력)
- 단점: 토큰 종량 과금, 파일 읽기/쓰기 등 도구를 직접 구현해야 함
- 적합: 정밀한 제어가 필요하거나, CC 시스템 프롬프트 노이즈가 품질에 영향을 줄 때

## 선택 가이드

```
코드 제어 필요 없음 (인터랙티브) → Plugin
코드 제어 + 구독 + 단순 호출     → claude -p
코드 제어 + 구독 + 스트리밍/세션  → Claude Agent SDK  ← 현재 fe-harness
코드 제어 + 완전 제어 + 노이즈 제로 → Anthropic SDK (종량)
```

## 참고: Ouroboros의 선택

Ouroboros는 `claude_agent_sdk` (Python)를 기본 백엔드로 사용.
구독 과금 + 코드 오케스트레이션 + 스트리밍 + 세션 resume을 활용.
추가로 LiteLLM 어댑터로 다른 모델도 지원 (OpenRouter 등).

## fe-harness 결정

- Plugin(A)과 독립 서비스(B) 두 가지로 구현하여 **컨텍스트 격리가 품질에 영향을 주는지** 비교
- B의 호출 방식: `claude -p` → `Claude Agent SDK`로 변경 (2026-03-31)
  - 근거: `claude -p`의 상위 호환. 스트리밍, 세션 resume 가능. 오버헤드 감소
