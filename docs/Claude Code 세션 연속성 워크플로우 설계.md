# Claude Code 세션 연속성 워크플로우 설계

> 세션 종료/중단 후 작업을 이어가기 위한 워크플로우 설계 정리
> 관련 노트: [[2026-02-15-session-wrap-플러그인-분석]]

---

## 문제 인식

| 문제 | 상황 |
|------|------|
| 세션 종료 후 AI가 컨텍스트를 잃음 | exit, crash, 세션 만료 |
| 여러 기능 동시 작업 시 AI가 맥락 혼동 | 여러 기능 병렬 작업 |
| context 꽉 찰 때 자동 관리 안 됨 | 긴 작업 세션 |
| 프로젝트 커지면 .ai/ 작업 내용 누적 | 장기 프로젝트 |

---

## 설계 원칙

- **가볍게, 파일 하나로** — 무거운 자동화 대신 단순한 패턴
- **한 번에 하나의 기능** — 여러 기능 동시 작업 시 git worktree로 격리
- **규모에 따라 스케일** — 단순 작업이면 20줄, 복잡하면 500줄, 같은 파일

---

## 콘텐츠 유형 3가지

```
1. 작업 문서 (current.md)     → 세션 이어가기 + 스펙/설계 (작업 단위)
2. 패턴 레퍼런스 (patterns/)  → 비슷한 작업 참고용 (장기, 점진적 개선)
3. 저장 위치 분류              → 성격별로 다른 곳에
```

### 1. 작업 문서 (.ai/current.md)

**하나의 파일이 규모에 따라 스케일:**

| 작업 규모 | current.md 크기 | 포함 내용 |
|----------|----------------|----------|
| 단순 (버그, 작은 기능) | ~20줄 | 목표 + 진행 + 다음 할 일 |
| 복잡 (멀티 세션 기능) | ~500줄 | 스펙 + 설계 결정 + 구현 단계 + 진행 |

- 세션 시작 시 **전체 읽기** (500줄도 Claude 컨텍스트 대비 작은 양)
- 작업 완료 시 삭제 or 패턴으로 승격

**템플릿**: `ai-ax/.ai/templates/simple-task.md`, `complex-task.md`

**라이프사이클**:
```
작업 시작      → current.md 생성 (템플릿 기반)
세션 중단      → /save로 현재 상태 저장
세션 재개      → current.md 읽고 이어서
작업 완료      → 삭제 or patterns/로 승격
```

### 2. 패턴 레퍼런스 (.ai/patterns/)

> 논의 중 — 아직 미확정

**핵심 특징**:
- 선택적 저장 (가치 있는 작업만)
- AI가 찾아서 참고 가능
- 비슷한 작업 시 재활용 (A 작업 참고해서 E 작업 진행)
- 반복하면서 점진적 개선 (A+E 경험으로 패턴 업데이트)

**예시 흐름**:
```
A 작업 완료 → 남길 가치 있음 → patterns/에 저장
B 작업 완료 → 패스
D 작업 완료 → 반복 패턴 → patterns/에 저장
E 작업 시작 → "A랑 비슷한데?" → A 패턴 참고해서 진행
E 완료 후   → A 패턴 개선 → 업데이트
```

**저장 위치**:
- 프로젝트 로컬: `{project-root}/.ai/patterns/`
- 글로벌 (프로젝트 무관): 미정 (파킹)

**모노레포 접근 문제**:
- 루트에서 Claude 실행 → `.ai/patterns/` 접근 가능
- 서비스에서 Claude 실행 → 상위 `.ai/` 안 보임
- 해결: 루트 CLAUDE.md에 패턴 위치 명시 → Claude가 계층적 CLAUDE.md 읽으므로 경로 인지 가능

### 3. 저장 위치 분류

```
.ai/current.md        ← 활성 작업 상태 (일시적, 덮어쓰기)
.ai/patterns/         ← 작업 레퍼런스 (장기, 선택적 누적)
.ai/templates/        ← 작업 문서 템플릿
obsidian-note/        ← 배움/지식 (영구, 개인)
{project}/CLAUDE.md   ← 프로젝트 규칙 (영구, 프로젝트)
~/.claude/MEMORY.md   ← 글로벌 패턴 (영구, 200줄 한도)
```

## git worktree = 컨텍스트 격리

```
~/project/              ← main
~/project-auth/         ← feature/auth (별도 디렉토리)
~/project-payment/      ← feature/payment (별도 디렉토리)

→ 각 worktree가 독립적 컨텍스트
→ Claude 열면 해당 worktree만 인식
→ 멀티 기능 컨텍스트 혼동 없음
```

## Claude memory 한계 인식

| 상황 | 신뢰도 |
|------|--------|
| 단일 세션, 단일 기능 | 높음 |
| 단일 세션, 여러 기능 | context 크기 따라 |
| **멀티 세션, 여러 기능** | 낮음 (git worktree로 해결) |

```
MEMORY.md       → 세션 시작 시 자동 로드, 200줄 한도
다른 memory 파일 → 자동 로드 안 됨, 명시적으로 읽어야 함
```
