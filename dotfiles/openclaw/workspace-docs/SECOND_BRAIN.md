# SECOND_BRAIN.md

## 목적
- 코드 작업은 Claude Code 중심으로 진행
- 개인 운영/기억/리마인드는 OpenClaw가 유지
- 결정/약속/다음 액션이 세션을 넘어 유지되게 설계

## 운영 원칙 (짧고 강하게)
1. 중요한 결정은 반드시 파일로 남긴다.
2. 당일 로그(`memory/YYYY-MM-DD.md`)와 장기 기억(`MEMORY.md`)을 분리한다.
3. 자동화는 작은 루틴부터 시작하고, 1주 단위로 늘린다.

## 파일 구조
- `memory/YYYY-MM-DD.md`: 하루 작업 로그/결정/메모
- `MEMORY.md`: 장기적으로 유지할 사실/선호/반복 패턴
- `HEARTBEAT.md`: 주기 실행 체크리스트(짧게)
- `SECOND_BRAIN.md`: 운영 규칙(이 문서)

## 기록 포맷
### Daily 로그 템플릿 (`memory/YYYY-MM-DD.md`)
- 오늘 목표
- 결정 사항
- 진행 중 이슈
- 다음 액션 (최대 5개)
- 나중에 MEMORY.md로 올릴 후보

### 장기 기억 승격 기준 (`MEMORY.md`)
- 반복되는 선호/작업 방식
- 다음에도 재사용할 의사결정 기준
- 일정/협업/건강/생활 패턴 중 지속 가치가 큰 것

## 1주차 자동화 (최소 세팅)
- [ ] 오전 1회: 오늘 일정/할 일 브리프
- [ ] 오후 1회: 진행 점검 + 막힌 이슈 확인
- [ ] 밤 1회: 데일리 로그 정리 + 내일 첫 액션 생성
- [ ] 2~3일 간격: daily 로그에서 MEMORY 승격 후보 추출

## 코딩 워크플로 연동
1. 구현은 Claude Code에서 수행
2. 결과를 OpenClaw에 짧게 보고
3. OpenClaw가 `memory/YYYY-MM-DD.md`에 요약 저장
4. 주기적으로 MEMORY.md 업데이트

## 실패 방지 규칙
- 자동화가 길어지면 실패한다. 체크리스트는 5줄 이하 유지.
- 한 번에 전부 자동화하지 않는다. 루틴 1개씩 추가한다.
- "기억했겠지" 금지. 반드시 파일에 쓴다.
