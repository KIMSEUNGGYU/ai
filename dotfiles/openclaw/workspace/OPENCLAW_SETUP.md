# OpenClaw 초기 셋업 체크리스트 (gyu용)

## 0) 지금 상태 점검
- [ ] `openclaw status`로 런타임/모델/채널 확인
- [ ] 기본 응답 톤이 의도대로 나오는지 테스트

## 1) 페르소나/사용자 컨텍스트 고정
- [x] `SOUL.md` (말투/원칙)
- [x] `AGENTS.md` (세션 시작/행동 규칙)
- [x] `USER.md` (호칭/타임존/노트 경로)
- [ ] 자주 쓰는 표현/금지 표현 3~5개 추가

## 2) 메모리 체계 구축
- [x] `MEMORY.md` 생성 (장기 기억)
- [x] `memory/_TEMPLATE.md` 생성 (일일 템플릿)
- [ ] 오늘 로그 파일 생성: `memory/YYYY-MM-DD.md`
- [ ] 중요한 결정은 장기 기억에도 중복 기록

## 3) 스킬 운영(AgentSkills)
- [ ] 반복 작업 3개 정의 (예: GitHub, 문서 정리, 릴리즈)
- [ ] 각 작업을 스킬로 분리할지 판단
- [ ] 스킬별 입력/출력 포맷 고정
- [ ] 실패 시 fallback 규칙 작성

## 4) 자동화 (Heartbeat/Cron)
- [x] `HEARTBEAT.md` 기본 정책 유지
- [ ] 주기 점검 필요 시 작업만 추가
  - 예: "매일 09:00 미완료 TODO 요약"
  - 예: "매주 월요일 의존성 업데이트 후보 점검"

## 5) 서브에이전트 분업 규칙
- [ ] 단순 수정(1~3파일): 메인 세션 직접 처리
- [ ] 중간 이상 작업: `sessions_spawn`으로 분리
- [ ] 완료 기준(Definition of Done) 먼저 명시

## 6) 추천 기본 루틴
- 아침: 오늘 목표 3개 기록
- 작업 중: 결정/실수/재발방지 포인트 즉시 메모
- 마감: `memory/YYYY-MM-DD.md` 요약 5줄 + 장기기억 반영

## 7) 빠른 시작 명령 예시
```bash
openclaw status
openclaw help
```

---

필요하면 다음 단계로:
1) 너 전용 스킬 1개 설계
2) heartbeat 실전 태스크 1개 등록
3) 메모리 자동 기록 포맷 고정
