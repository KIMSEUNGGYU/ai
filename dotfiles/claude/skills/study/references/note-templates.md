# 학습 노트 템플릿

## ~/hq/ 학습 노트 공통 형식

딥다이브나 커리큘럼에서 ~/hq/로 내보낼 때의 공통 형식:

```markdown
---
tags: [learn, {주제 태그}]
date: {YYYY-MM-DD}
source: {출처 — 강의명, 레포 URL, 또는 "AI 학습"}
---

# {주제}

## 핵심 발견

- {배운 것 1} — {왜 중요한지 한 줄}
- {배운 것 2} — {왜 중요한지 한 줄}

## 상세

### {발견 1}

{설명}

**왜?**
{근거, 이유, 원리}

{코드 예시 (필요시)}

## 연관 키워드

- {관련 개념/기술}
```

## 강의 상세 노트 (학습용)

파일명: `{섹션번호}-{순번} {주제명}.md`

```markdown
---
tags: [learn, {강의 태그}]
date: {YYYY-MM-DD}
source: {강의명}
section: {섹션번호}
---

# {섹션번호}-{순번}. {주제명}

## 왜 필요한가?
{동기 설명}

## 핵심 개념
{개념 설명}

## 문법/구문
{기본 문법 코드 블록}

## 실습
{실습 코드 + 결과}

## 실무 팁
{강의에서 알려준 실무 팁}

## 정리
{핵심 포인트 3-5개 bullet}
```

## 강의 정리 노트 (복습용)

파일명: `{섹션번호} {섹션명} 정리.md`

```markdown
---
tags: [learn, {강의 태그}, summary]
date: {YYYY-MM-DD}
source: {강의명}
---

# {섹션번호}. {섹션명} 정리

## {소주제 1}
{bullet 형태로 핵심만}

## {소주제 2}
{bullet 형태로 핵심만}
```

## 정리 스타일 규칙

1. **개념 → 문법 → 실습 → 팁 → 정리** 순서 (자료 기반 학습)
2. **핵심 포인트**는 `>` blockquote로 강조
3. **비교 분석**은 표 형식
4. **실행 순서** 같은 프로세스는 번호 리스트
5. **Q&A 형식**으로 "왜?" 질문에 답하는 구조 활용
6. 나중에 노트만 열면 복습할 수 있어야 한다
